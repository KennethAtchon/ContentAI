# Editor Rearchitecture Worklog

> **Parent LLD Index:** [../README.md](../README.md)
> **Purpose:** Capture research notes, codebase findings, open questions, and decision records while the phase LLDs stay focused and stable.

Use this folder for working material that should not clutter the phase documents. When a finding becomes an accepted design choice, summarize it in the relevant phase LLD and keep the detailed trail here.

## Files

| File | Purpose |
|---|---|
| [phase-0-findings.md](./phase-0-findings.md) | Raw/current findings for Phase 0 inventory and contract freeze. |
| [phase-1-findings.md](./phase-1-findings.md) | Raw/current findings for Phase 1 target schema coverage. |
| [decision-log.md](./decision-log.md) | Decision records with status, rationale, and consequences. |
| [research-notes.md](./research-notes.md) | External/internal research notes and source links. |

## Worklog Rules

- Keep findings factual and source them to a file path, code symbol, or URL.
- Mark decisions as `Proposed`, `Accepted`, `Rejected`, or `Superseded`.
- Do not treat worklog notes as implementation authority until they are promoted into an LLD.
- Prefer small dated entries over rewriting history.
