## Explanations For Skipped Questions

### Q9 — Sequence Trace (Model Answer)

On initial load to `/feed/123`, the likely high-level flow is:

1. **Root config loads first**  
   The browser loads the shell/root-config and single-spa setup.
2. **Activity functions are evaluated**  
   single-spa checks URL and determines active apps:
   - `navbar` active (`/`)
   - `feed` active (`/feed`)
   - `profile` inactive (`/profile` only)
3. **Active app source code is requested**  
   `navbar` and `feed` loading functions/imports are triggered (often concurrently).
4. **Lifecycle execution**  
   Each active app runs lifecycle functions:
   - `bootstrap` (typically once per app lifetime),
   - `mount` (for this route activation).
5. **Inactive app behavior**  
   `profile` remains inactive (usually not mounted, and often not fetched unless prefetch rules exist).

**Concurrency nuance:** network fetches for active apps can happen in parallel, and whichever app finishes loading first may mount first.

### Q11 — Debug Plan (First 8 Checks in Order)

Given: "`@org/feed` works locally but fails in production with `LOADING_SOURCE_CODE`"

1. **Import map/root-config resolution**
   - Confirm `@org/feed` resolves to the expected production URL.
2. **Direct URL verification**
   - Open resolved URL directly; confirm `200` and JavaScript payload (not HTML fallback/error page).
3. **Network subresource inspection**
   - Check whether lazy chunks requested by the entry file return `404`/`403`/wrong host.
4. **Webpack `publicPath` verification**
   - Ensure deployed asset base matches chunk URL generation path.
5. **Artifact version alignment**
   - Ensure import map points to the same build version as referenced chunks.
6. **Lifecycle export contract check**
   - Confirm deployed app exports `bootstrap`, `mount`, `unmount` correctly.
7. **Shared/external dependency resolution**
   - Verify React/SystemJS/shared externals are available and compatible in production.
8. **Environment diff check (local vs prod)**
   - Compare base URLs, CDN path, CSP, auth headers, feature flags, and minification differences.

### Bonus — Architecture Challenge (Model Direction)

#### 1) Build strategy per microfrontend
- Independent webpack builds per team with content-hashed output.
- Route-level/feature-level code splitting inside each app.
- Bundle budgets and build-time performance gates in CI.

#### 2) Runtime orchestration strategy
- Central root config + import maps for app discovery/version routing.
- Clear activity functions with small overlap to avoid accidental over-mounting.
- Optional predictive prefetch only for high-likelihood next routes.

#### 3) Shared dependency/version policy
- Share heavy and stable dependencies (for payload wins) when compatibility can be enforced.
- Keep fast-changing dependencies app-local to reduce cross-team lockstep.
- Enforce compatibility windows and contract tests.

#### 4) Operational guardrails
- Per-app observability (mount latency, chunk load errors, lifecycle failures).
- Fast rollback via import-map pointer rollback.
- Explicit ownership/on-call boundaries per microfrontend.

**Major risk:** cross-app dependency drift causing runtime breakage.  
**Mitigation:** compatibility matrix, staged rollout, canary checks, automated integration smoke tests.
