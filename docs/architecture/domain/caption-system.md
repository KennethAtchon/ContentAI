# Caption System Architecture

The caption system spans two distinct domains that share the word "caption":

1. **Timeline Captions** — subtitles/overlays rendered on the video editor canvas (word-level tokens, styled presets, HTML5 canvas rendering).
2. **AI-Generated Post Captions** — Instagram post captions (and hooks/scripts) generated by LLMs via `POST /api/generation`.

Both are documented here.

---

## Part 1: Timeline Caption System

### Data Model

#### Token (atomic unit)
```typescript
interface Token {
  text: string
  startMs: number
  endMs: number
}
```

Tokens come from OpenAI Whisper (word-level timestamps) or manual input.

#### CaptionDoc (PostgreSQL table: `caption_doc`)

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID PK | |
| `user_id` | FK → users | Ownership/privacy |
| `asset_id` | FK → assets (nullable) | Linked audio asset |
| `language` | text | Default `"en"` |
| `tokens` | JSONB | `Token[]` array |
| `full_text` | text | Full transcript string |
| `source` | enum | `"whisper" \| "manual" \| "import"` |

Unique constraint: `(user_id, asset_id)` where `asset_id IS NOT NULL` — one transcription per asset.

#### CaptionPreset (PostgreSQL table: `caption_preset`)

Defines how captions look on screen. Stored as JSONB `definition: TextPreset`.

Key fields in `TextPreset`:
- `typography` — font family, weight, size, letter-spacing, line-height
- `layers` — ordered render layers: fill, stroke, shadow, background, glow
- `layout` — alignment (left/center/right), `maxWidthPercent`, `positionY` (0–100)
- `entryAnimation` / `exitAnimation` / `wordActivation` — animation definitions
- `groupingMs` — max duration per "page" of tokens (default 1400ms)
- `exportMode` — `"full" | "approximate" | "static"`

**10 seeded built-in presets:** `hormozi`, `clean-minimal`, `dark-box`, `karaoke`, `bold-outline`, `pop-scale`, `slide-up`, `fade-scale`, `glitch`, `word-highlight-box`.

#### CaptionClip (timeline object)

A `CaptionClip` is a clip on the editor timeline that references a `CaptionDoc`:

```typescript
interface CaptionClip extends BaseClip {
  type: "caption"
  captionDocId: string
  originVoiceoverClipId: string | null
  sourceStartMs: number    // window into the caption doc tokens
  sourceEndMs: number
  stylePresetId: string
  styleOverrides: {
    positionY?: number
    fontSize?: number
    textTransform?: "none" | "uppercase" | "lowercase"
  }
  groupingMs: number
}
```

### Backend

#### API Endpoints (`/api/captions`)

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/transcribe` | Transcribe audio asset via Whisper (rate-limited: 2/min, 20/hr) |
| `POST` | `/manual` | Create a caption doc from user-provided tokens |
| `GET` | `/doc/:captionDocId` | Fetch a single caption doc |
| `PATCH` | `/doc/:captionDocId` | Update tokens/text of a caption doc |
| `GET` | `/:assetId` | Get caption doc linked to an asset |
| `GET` | `/presets` | List all caption presets |

Route: `backend/src/routes/editor/captions.ts`  
Service: `backend/src/domain/editor/captions.service.ts`  
Repository: `backend/src/domain/editor/captions.repository.ts`

#### Transcription Flow (Whisper)

```
POST /api/captions/transcribe { assetId, force? }
  ↓
CaptionsService.transcribeAsset()
  ├── Validate asset exists and is audio type
  ├── Download from R2 (enforces 25MB limit)
  ├── Call OpenAI Whisper API
  │     model: "whisper-1"
  │     response_format: "verbose_json"
  │     timestamp_granularities: ["word"]
  └── Upsert into caption_doc (source: "whisper")
  ↓
Returns: { captionDocId, tokens, fullText }
```

#### Orphan Cleanup

When an editor project is saved, `EditorService` (`backend/src/domain/editor/editor.service.ts`) extracts all `captionDocId` values from the timeline, compares with the previous version, and calls `cleanupCaptionDocsIfUnreferenced()` — which deletes any caption docs no longer referenced in any of the user's projects.

### Frontend

All caption frontend code lives in `frontend/src/features/editor/caption/`.

#### Data Hooks

| Hook | Query Key | Purpose |
|------|-----------|---------|
| `useCaptionDoc(id)` | `["api","captions","doc",id]` | Fetch caption doc + tokens |
| `useCaptionPresets()` | `["api","captions","presets"]` | Fetch all presets |
| `useUpdateCaptionDoc()` | mutation | PATCH tokens/fullText |
| `useTranscription()` | mutation | POST to trigger Whisper |
| `useCaptionCanvas(...)` | — | Real-time canvas rendering hook |

#### Rendering Pipeline

Each frame rendered in the editor preview goes through four stages:

```
1. sliceTokensToRange(tokens, sourceStartMs, sourceEndMs)
      → filters tokens to clip's source window; re-zeros timing

2. buildPages(tokens, groupingMs, gapThresholdMs=800ms)
      → groups tokens into CaptionPage[] by timing gaps and max span

3. computeLayout(ctx, page, preset, canvasW, canvasH)
      → font scaling, text wrapping, absolute token positions (x, y, w)
      → returns CaptionLayout with block dimensions and line structure

4. renderFrame(ctx, layout, relativeMs, preset)
      → applies entry/exit animations (opacity, scale, translate)
      → applies wordActivation effects (state: upcoming/active/past)
      → draws each layer in order: background → stroke → fill → shadow → glow
```

Files: `page-builder.ts`, `layout-engine.ts`, `renderer.ts`, `slice-tokens.ts`, `easing.ts`

#### Inspector Components

| Component | Purpose |
|-----------|---------|
| `CaptionPresetPicker` | Select style preset; warns on export mode |
| `CaptionStylePanel` | Adjust groupingMs, positionY, fontSize, textTransform |
| `CaptionTranscriptEditor` | Edit fullText and save to backend |
| `CaptionLanguageScopeNotice` | English-only limitation notice |

#### Reducer Actions

Defined in `frontend/src/features/editor/model/editor-reducer-clip-ops.ts`:

- **`ADD_CAPTION_CLIP`** — creates a new `CaptionClip` on a track; requires `durationMs > 0` and `sourceStartMs < sourceEndMs`
- **`UPDATE_CAPTION_STYLE`** — merges `presetId`, `styleOverrides`, and/or `groupingMs` into an existing caption clip

## Part 2: AI-Generated Post Captions (`/api/generation`)

This is the LLM generation system. It produces Instagram post captions, video hooks, and script notes from an analyzed source reel.

### Endpoint

```
POST /api/generation
Body: { sourceReelId, prompt, outputType }
```

`outputType` is one of:
- `"hook_only"` — generates 5 hook variations
- `"caption_only"` — generates hook + caption
- `"full_script"` — generates hook + caption + script bullet points

Route: `backend/src/routes/generation/index.ts`  
Logic: `backend/src/domain/content/source-reel-generation.ts`

### Generation Flow

```
POST /api/generation { sourceReelId, prompt, outputType }
  ↓
contentService.generateFromSourceReel()
  ↓
generateContentFromSourceReel()
  ├── Fetch reel row + reel_analysis from DB
  │     (hookPattern, hookCategory, emotionalTrigger, formatPattern, ctaType, captionFramework)
  │
  ├── if outputType === "hook_only":
  │     system prompt: hook-writer.txt
  │     user content: niche, hook pattern, emotional trigger, user prompt
  │     → callAi({ modelTier: "analysis", maxTokens: 512 })
  │     → parse JSON string[] → { hook: hooks[0], scriptNotes: hooks[1..] }
  │
  └── else (caption_only / full_script):
        system prompt: remix-generation.txt
        user content: full analysis + original hook (for structure, not copying) + user prompt
        → callAi({ modelTier: "generation", maxTokens: 1024 })
        → parse JSON { hook, caption, scriptNotes }
  ↓
content.createReelGeneratedDraftWithQueueEnrollment()
  → inserts into generated_content table
  → returns new draft with queue row
```

### AI Client & Provider System

`callAi()` in `backend/src/lib/aiClient.ts` routes to `callAiWithFallback()` which selects a provider dynamically.

#### Providers (`backend/src/lib/ai/providers.ts`)

| Provider ID | SDK | Notes |
|-------------|-----|-------|
| `openrouter` | `@ai-sdk/openai` (OpenRouter base URL) | Default first priority |
| `openai` | `@ai-sdk/openai` | Second priority |
| `claude` | `@ai-sdk/anthropic` | Third priority (Anthropic Claude) |

**Priority order** defaults to `[openrouter, openai, claude]` but is overridable at runtime via the `system_config` DB table (`ai.provider_priority`).

#### Model Tiers

Each provider supports two tiers:

| Tier | Usage | Default intent |
|------|-------|---------------|
| `"analysis"` | hook_only, reel analysis | cheaper/faster model |
| `"generation"` | caption_only, full_script, chat | smarter model |

Model names per tier are configurable via `system_config` DB keys (e.g. `claude_analysis_model`, `claude_generation_model`) with ENV var fallbacks.

#### Prompts

| File | Used For |
|------|---------|
| `prompts/hook-writer.txt` | `hook_only` — returns JSON string array of 5 hooks |
| `prompts/remix-generation.txt` | `caption_only` / `full_script` — returns `{ hook, caption, scriptNotes }` |
| `prompts/reel-analysis.txt` | Reel analysis pipeline (separate feature) |
| `prompts/chat-generate.txt` | Chat/streaming generation (separate feature) |

## How the Two Caption Systems Relate

The two systems are **independent** — they share the word "caption" but serve different purposes:

| Aspect | Timeline Captions | AI Post Captions |
|--------|-------------------|-----------------|
| What it is | Subtitle overlay on video canvas | Instagram post caption text |
| Storage | `caption_doc` table (tokens + timing) | `generated_content` table (text fields) |
| AI provider | OpenAI Whisper (speech-to-text) | Any LLM via provider registry |
| Entry point | `POST /api/captions/transcribe` | `POST /api/generation` |
| Output | `Token[]` with ms timestamps | `{ hook, caption, scriptNotes }` strings |
| Editor use | Rendered as styled subtitle overlay | Displayed in content detail / queue UI |

The only indirect link: a user might use an AI-generated `caption` text as the basis for a manually-entered caption doc, but there is no automatic bridge between them in the current codebase.
